# NexaPress Access Guard

NexaPressのログインページと管理画面へのアクセスを保護する、軽量なセキュリティ拡張機能です。

NexaPressへの基本同梱を想定していますが、拡張機能単体でもインストールできます。

## バージョン

```text
1.0.0
```

## 対応環境

- NexaPress 1.9.0以上
- PHP 8.0以上
- Apache
- PHP ZipArchive

## 主な機能

### ログインページアクセス制限

許可IPに登録されたIPアドレスだけが、ログインページへアクセスできるようにします。

- 有効・無効の切り替え
- 許可IP以外からのアクセスを拒否
- 現在のIPが未登録の場合は有効化を防止

### 管理ページアクセス制限

許可IPに登録されたIPアドレスだけが、`/admin`以下の管理ページへアクセスできるようにします。

- 有効・無効の切り替え
- 許可IP以外からのアクセスを拒否
- 現在のIPが未登録の場合は有効化を防止

### ログインページ変更

標準のログインURLを任意のURLへ変更します。

標準URL：

```text
/admin/login
```

変更例：

```text
/secure-login
```

機能を有効にすると、元の`/admin/login`は404になります。

### ログインロック

同じIPアドレスからのログイン失敗が指定回数に達した場合、一時的にログインをロックします。

選択できる失敗回数：

- 3回
- 5回
- 10回

選択できるロック期間：

- 5分
- 15分
- 30分
- 1時間
- 6時間
- 24時間

ログインに成功すると、そのIPアドレスの失敗回数はリセットされます。

### フェールワンス

許可IPに登録されていないIPアドレスから正しいログイン情報が送信された場合でも、指定回数だけ意図的にログインを失敗させます。

選択できる回数：

- 1回
- 2回
- 3回

許可IPからのログインでは発動しません。

### 許可IP

信頼するIPアドレスを登録します。

許可IPは次の機能で使用されます。

- ログインページアクセス制限
- 管理ページアクセス制限
- フェールワンスの対象外

現在接続しているIPアドレスは、設定画面上部に表示されます。

### 拒否IP

登録されたIPアドレスから、ログインページと管理ページへのアクセスを拒否します。

現在接続しているIPアドレスは、誤操作防止のため拒否IPへ登録できません。

## 設定画面

すべての設定は、1つの管理画面に表示されます。

各機能は個別のフォームになっているため、保存もそれぞれ独立しています。

```text
ログインページアクセス制限
├─ 有効・無効
└─ この設定を保存

管理ページアクセス制限
├─ 有効・無効
└─ この設定を保存

ログインページ変更
├─ 有効・無効
├─ 変更後URL
└─ この設定を保存

ログインロック
├─ 有効・無効
├─ 失敗回数
├─ ロック期間
└─ この設定を保存

フェールワンス
├─ 有効・無効
├─ 失敗させる回数
└─ この設定を保存

許可IP
├─ IP追加
└─ 登録済みIPの削除

拒否IP
├─ IP追加
└─ 登録済みIPの削除
```

## インストール方法

1. 配布用ZIPを用意します。
2. NexaPressの管理画面へログインします。
3. 「拡張機能」を開きます。
4. ZIPファイルをアップロードします。
5. 「NexaPress Access Guard」を有効化します。
6. 拡張機能メニューから「アクセスガード」を開きます。

## 安全な初期設定手順

管理画面へアクセスできなくなる事故を防ぐため、次の順番で設定してください。

1. NexaPress Access Guardを有効化する
2. 設定画面を開く
3. 画面上部に表示された現在のIPを確認する
4. 現在のIPを「許可IP」へ追加する
5. ログインロックを設定する
6. フェールワンスを設定する
7. ログインページ変更を設定する
8. ログインページアクセス制限を有効にする
9. 管理ページアクセス制限を最後に有効にする

ログインページを変更した場合は、変更後のURLを必ず控えてください。

## 設定データの保存場所

設定やログインロック情報は、拡張機能フォルダ外へ保存されます。

```text
storage/
└─ extensions/
   └─ access-guard/
      ├─ settings.json
      ├─ login-locks.json
      └─ fail-once.json
```

そのため、拡張機能本体を更新しても設定データは維持されます。

## 配布用ZIPの作成

プロジェクトフォルダへ移動します。

```bat
cd /d D:\github作業フォルダ\nexapress-access-guard
```

PHPファイルを一括で構文チェックします。

```bat
for /R %f in (*.php) do @php -l "%f"
```

すべてのPHPファイルで次のように表示されることを確認します。

```text
No syntax errors detected
```

その後、配布用ZIPを作成します。

```bat
php build.php
```

正常に完了すると、次のZIPが作成されます。

```text
dist\nexapress-access-guard-1.0.0.zip
```

作成されたZIPを確認します。

```bat
dir dist
```

## ZIP内部の構成

```text
access-guard/
├─ manifest.json
├─ bootstrap.php
├─ README.md
├─ admin/
├─ includes/
└─ assets/
```

`build.php`と`dist`フォルダは、配布用ZIPには含まれません。

## 拡張機能の更新について

`manifest.json`には、将来の拡張機能更新機能で使用する更新元情報を含めています。

```json
"update": {
  "provider": "github",
  "repository": "aki-sho/nexapress-access-guard",
  "asset": "nexapress-access-guard-{version}.zip"
}
```

現在のNexaPress本体には、拡張機能を個別に自動更新する処理はまだありません。

今後、本体側へ拡張機能更新機能を追加した際に、GitHub Releasesから更新情報と配布用ZIPを取得できる構成にしています。

## 注意事項

### IPアドレスについて

現在のバージョンでは、接続元IPとしてPHPの`REMOTE_ADDR`を使用します。

リバースプロキシ、CDN、ロードバランサーを使用している環境では、実際の利用者ではなくプロキシ側のIPが表示される場合があります。

### 動的IPについて

インターネット接続環境によっては、IPアドレスが定期的に変更されます。

IPアドレスが変わる環境で管理ページアクセス制限を有効にすると、管理画面へ入れなくなる可能性があります。

### ログインURLについて

ログインページ変更を有効にした場合、標準の`/admin/login`は使用できません。

変更したURLを忘れないように保管してください。

### テストについて

本番環境で有効にする前に、ローカル環境または検証環境で動作を確認してください。

特に次の機能は、必ず個別に確認してください。

- ログインページ変更
- ログインページアクセス制限
- 管理ページアクセス制限
- ログインロック
- フェールワンス
- 許可IP
- 拒否IP

管理ページアクセス制限は、必ず最後に確認してください。
